MVVM for JavaScript

The ExtJs and Sencha Touch libraries advocate the usage of an MVC design pattern which does not do enough to segregate business logic from the view. This causes difficulty in testing, as well as difficulty in maintenance as your application grows in size and complexity. GluJs attempts to remedy this by pushing all business logic to the model-view layer through a two-way binding mechanism with the view. 
                          
                               THE VIEW:
            examples.transactions.TransactionView = {
                xtype: 'fieldset',
                height:'100%',
                layout:'border',
                items:[
                    {
                        name: 'transactionsStore',
                        stripeRows: true,
                        xtype:'grid',
                        region:'center',
                        selections: '@{transactionSelections}',
                        bbar: {
                            xtype:'paging',
                            store: '@{transactionsStore}',
                            displayInfo: true,
                            displayMsg: 'Displaying transactions {0} - {1} of {2}',
                            emptyMsg: "No transactions to display",
                            paging: '@{transactionPaging}'
                        }
                    },
                    {
                        xtype:'form',
                        width: 500,
                        region:'east',
                        items:[
                            {
                                name:'testLabel',
                                xtype:'label'
                            },
                            {
                                name: 'selectedRecord',
                                xtype: 'label'
                            }

                        ]
                    }
                ]
            }
                              
                          
                      

                      THE VIEW-MODEL:
            examples.transactions.TransactionViewModel = {
                testLabel: 'Details for Selected Transaction',
                transactionPaging:{start:0, limit:25},
                transactionSelections: [],
                selectedRecord: 'No Record Selected',
                transactionsStore:{
                    mtype:'store',
                    idProperty: 'id',
                    recType:'transaction',
                    root: 'data',
                    totalProperty: 'totalRecords',
                    url: '/readtransactions',
                    params: function(){
                        return this.transactionPaging
                    }
                },
                setTransactionSelections: function(newValue, oldValue) {
                    Glu.log.info('setTransactionSelections');
                    if (Glu.isArray(newValue) && newValue.length > 0 && newValue[0].data) {
                        this.setSelectedRecord(Glu.json.stringify(newValue[0].data));
                    }
                },
                setTransactionPaging: function(newValue, oldValue) {
                    this.set('transactionPaging', newValue);
                    this.transactionsStore.load();
                }
            };
GluJs also provides enhanced automagic features to form natural linkages between model, view and view-model segregations. This results in smaller, more consistent and more intuitive code.
